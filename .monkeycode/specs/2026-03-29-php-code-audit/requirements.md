# PHP 代码审计系统需求文档

需求名称：php-code-audit
更新日期：2026-03-29

## 1. 概述与目标

开发一套 PHP 代码自动化审计系统，专注于深度污点分析，精准识别高危漏洞（CVSS ≥ 6.0），极力避免低危误报。系统面向安全工程师和开发团队，提供可配置的漏洞扫描和详细的漏洞报告。

## 2. 功能范围

### 2.1 核心审计能力

| 漏洞类型 | CVSS 范围 | 审计目标 |
|---------|----------|---------|
| 远程代码执行 (RCE) | 9.0~10.0 | eval, assert, system, exec, shell_exec, preg_replace(/e), create_function, 反引号 |
| SQL 注入 | 7.5~9.8 | mysql_query, mysqli_query, PDO::query, whereRaw, 字符串拼接 |
| 任意文件写入 | 7.0~9.8 | file_put_contents, fwrite, move_uploaded_file |
| 文件包含 | 7.0~9.8 | include, include_once, require, require_once |
| 反序列化漏洞 | 8.0~9.8 | unserialize, POP 链, 魔术方法 |

### 2.2 污点传播追踪

- 识别外部输入源：`$_GET`, `$_POST`, `$_REQUEST`, `$_COOKIE`, `$_FILES`, `php://input`, `getenv`, `getallheaders`
- 追踪数据流经函数调用、数组赋值、对象传递的变化
- 识别过滤节点：`intval`, `htmlspecialchars`, `addslashes`, 白名单校验
- 检测过滤绕过：宽字节注入、str_replace 递归绕过

### 2.3 CVSS 评估

- 计算攻击向量（网络/本地）
- 评估攻击复杂度（低/高）
- 判断权限需求（无/低）
- 确定影响范围（完全/部分）

### 2.4 扫描配置

- CVSS 阈值过滤（默认 6.0）
- 扫描深度配置
- 目录/文件排除规则

## 3. 输出要求

### 3.1 漏洞报告格式

```
## 🚨 高危漏洞报告

### 1. 漏洞定位
- **漏洞类型**: [类型]
- **CVSS 预估**: [分数]
- **受影响文件**: [文件:行号]
- **危险函数**: [函数]

### 2. 污点分析
- **污点源**: [输入源]
- **传播路径**: 数据流向描述
- **过滤失效**: 绕过原因

### 3. 漏洞证明
- 利用代码或 HTTP 请求示例
```

### 3.2 扫描统计

- 扫描文件总数
- 发现漏洞数量（按类型分类）
- 高危/中危分布

## 4. 约束条件

1. **准确性优先**：未发现高危漏洞时，输出"未发现 CVSS ≥ 6.0 的高危漏洞"
2. **禁止报告**：信息泄露、反射型 XSS、CSRF、路径泄露、弱口令等低危问题
3. **语言**：专业中文输出，代码注释使用英文
